## スキル概要
- インフラ: 基本的にAWS CDKで構築するケースが多いです（TerraformによるGoogle Cloud構築経験もあります）
  - 実行環境: Lambda（コスト面からECS Fargate等よりLambdaを採用するケースが多いです）
  - API: LambdaをバックエンドにAppSyncあるいはAPI Gatewayを使うことが多いです。非同期処理でSQS + DynamoDBも使います（Webhookの処理等）
  - 認証: Cognito、Firebase（ログイン手段の選択肢の多さから、認証部分のみAWSではなくFirebaseを使うことがあります）
  - ストレージ: S3（バケット内で境界を設けてのマルチテナント対応や、アップロードをトリガーにした画像変換をよく実装します）
  - 静的サイトホスティング: S3 + Cloudfront
  - その他AWSでよく使うサービス: Step Functions + EventBridge（バッチ処理）、Route53、Amazon SES、IAM Identity Center（開発者招待）、AWS WAFなど
- バックエンド: TypeScript + DDDでの実装が多いです
  - TypeScript: ユニオン型などを積極的に使い、動かす前にエラーに気付くコーディングを心がけています
  - DDD: 要件を議論しながらドメイン層から書いていくことが多いです。Value ObjectやEntityは、TypeScriptを活かしてクラスベースではなく型ベースで書くスタイルです
- フロントエンド: Vue.js + Tailwind CSSを使うことが多いです
- iOS/Androidアプリ: Swiftの経験はありますが、最近はIonicのようなウェブ技術によるアプリ開発を多用していました
- UIデザイン: 最近はデジタル庁のガイドライン・UIコンポーネントに準拠しての構築が多いです
- AI開発: LLMによる判断ループ、評価者LLMを用いたLLM-as-a-Judge構成、人間レビューの永続化による継続的精度改善等のアプローチでエージェントを開発しています
